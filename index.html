NODE 04 · HOW I THINK ABOUT THIS

Four ideas behind most of my work

The projects above are all built from the same handful of underlying patterns. Here's what each one actually means in practice.

RAG
Retrieval-Augmented Generation

Instead of asking a model to answer from memory, I fetch the actual relevant documents first — hybrid keyword + vector search, reranked with a cross-encoder — so the answer is grounded in something real, with a dashed retry line back to retrieval when the answer isn't well supported.

Hybrid RetrievalRerankingGrounding
Agentic Workflows
Routing, not one giant prompt

A router classifies the task and fans it out to specialized worker agents running in parallel — research, extraction, drafting — then a reducer merges their output into one result. It's slower to design than a single prompt, and far more reliable in production.

LangGraphParallel WorkersHuman-in-the-Loop
Deployment
From notebook to something that stays up

A model that only runs in a notebook isn't shippable. I containerize with Docker, wire up GitHub Actions for auto-deploy, and have taken the same discipline down to edge hardware — Edge AI devices — where compute and power are both tight.

DockerGitHub ActionsAWS EC2Edge
Multimodal Systems
Text, voice, and video in one pipeline

Real users don't type structured queries — they speak, upload PDFs, or point a camera. I've built Whisper-based STT into agent graphs, run inference on live camera feeds, and kept text, voice, and vision behind the same underlying pipeline so adding a modality doesn't mean rebuilding the system.
